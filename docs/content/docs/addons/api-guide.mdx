# API利用ガイド（アドオン開発者向け）

MoripaFishingは、アドオン開発者がプラグインの機能を拡張できるAPIを提供しています。
このガイドでは、APIの導入方法から基本的な使い方までを解説します。

## 依存関係の追加

### リポジトリの設定

MoripaFishing APIは[Maven Central](https://central.sonatype.com/artifact/party.morino/moripafishing-api)で公開されています。
認証不要で利用できます。

#### Gradle (Kotlin DSL)

```kotlin title="build.gradle.kts"
repositories {
    mavenCentral()
}

dependencies {
    compileOnly("party.morino:moripafishing-api:<version>")
}
```

#### Gradle (Groovy)

```groovy title="build.gradle"
repositories {
    mavenCentral()
}

dependencies {
    compileOnly 'party.morino:moripafishing-api:<version>'
}
```

#### Maven

```xml title="pom.xml"
<dependencies>
    <dependency>
        <groupId>party.morino</groupId>
        <artifactId>moripafishing-api</artifactId>
        <version>VERSION</version>
        <scope>provided</scope>
    </dependency>
</dependencies>
```

### SNAPSHOTビルド（JitPack）

開発中のブランチのAPIを利用したい場合は、[JitPack](https://jitpack.io/#morinoparty/MoripaFishing)経由で`<branch>-SNAPSHOT`またはコミットハッシュを指定できます。

```kotlin title="build.gradle.kts"
repositories {
    maven("https://jitpack.io")
}

dependencies {
    // ブランチの最新を追跡
    compileOnly("com.github.morinoparty.MoripaFishing:api:main-SNAPSHOT")
    // 特定のコミットを指定
    compileOnly("com.github.morinoparty.MoripaFishing:api:<commit-hash>")
}
```

:::warning[注意]
SNAPSHOTビルドやコミットハッシュ指定は開発中のため不安定です。本番環境ではリリースタグのバージョンを使用してください。
:::

### plugin.yml の設定

プラグインの`plugin.yml`に依存関係を追加してください。

```yaml title="plugin.yml"
depend:
  - MoripaFishing
```

## APIの取得

APIの取得口は`MoripaFishingAPIProvider`です。コア（`MoripaFishing`）が有効化時に登録し、無効化時に解除します。
プラグイン名でのキャスト（`getPlugin("MoripaFishing") as MoripaFishingAPI`）には依存しないでください。

### Kotlin

```kotlin
import party.morino.moripafishing.api.MoripaFishingAPIProvider

// コア未初期化時は IllegalStateException
val api = MoripaFishingAPIProvider.get()

// null 許容で取得したい場合
val apiOrNull = MoripaFishingAPIProvider.getOrNull()
```

### Java

```java
import party.morino.moripafishing.api.MoripaFishingAPI;
import party.morino.moripafishing.api.MoripaFishingAPIProvider;

MoripaFishingAPI api = MoripaFishingAPIProvider.get();
```

### ServicesManager 経由の取得

同じインスタンスがBukkitの`ServicesManager`にも登録されているため、そちらから取得することもできます。

```kotlin
val api = Bukkit.getServicesManager().load(MoripaFishingAPI::class.java)
    ?: error("MoripaFishing is not loaded")
```

## 利用可能なマネージャー

APIインスタンスから以下のマネージャーを取得できます。

| マネージャー | メソッド | 説明 |
|---|---|---|
| `ConfigManager` | `getConfigManager()` | プラグイン設定の取得・変更 |
| `FishManager` | `getFishManager()` | 魚の登録・取得 |
| `WorldManager` | `getWorldManager()` | 釣り場ワールドの管理 |
| `AnglerManager` | `getAnglerManager()` | つりびと（プレイヤー）データの管理 |
| `RarityManager` | `getRarityManager()` | レアリティの管理 |
| `RandomizeManager` | `getRandomizeManager()` | 魚・天候のランダム化 |
| `LogManager` | `getLogManager()` | ログ出力 |
| `PluginDirectory` | `getPluginDirectory()` | プラグインディレクトリの取得 |

また、APIインスタンス自体が天候ソースの登録口（`registerWeatherSource`など）を持ちます。詳細は[天候ソースの登録](#天候ソースの登録)を参照してください。

## 使用例

### 魚の一覧を取得する

```kotlin
val fishManager = api.getFishManager()
val allFish = fishManager.getFish()
allFish.forEach { fish ->
    println("${fish.id}: ${fish.displayName}")
}
```

### レアリティ別に魚を取得する

```kotlin
val fishManager = api.getFishManager()
val rarityId = RarityId("legendary")
val legendaryFish = fishManager.getFishesWithRarity(rarityId)
```

### つりびとの情報を取得する

```kotlin
val anglerManager = api.getAnglerManager()
val angler = anglerManager.getAnglerByMinecraftUniqueId(player.uniqueId)
angler?.let {
    println("Name: ${it.getName()}")
    println("World: ${it.getWorld()?.getId()}")
}
```

### ワールドの一覧を取得する

```kotlin
val worldManager = api.getWorldManager()
val worldIds = worldManager.getWorldIdList()
worldIds.forEach { id ->
    val world = worldManager.getWorld(id)
    println("World: $id")
}
```

### ワールドを作成する

ワールド生成の実体は [`MoripaFishing-Integration-WorldLifecycle` Integration](../integration/world-lifecycle) が提供します。
未導入環境では`createWorld`が`false`を返し、`getGeneratorIds()`は空リストを返します。

```kotlin
val worldManager = api.getWorldManager()
val generatorIds = worldManager.getGeneratorIds()
val created = worldManager.createWorld(FishingWorldId("my_world"), "terra")
```

## 天候ソースの登録

アドオンは`WeatherSource`を実装して独自の天候ソースを提供できます。
登録したキーを設定の[`weatherSource`](/docs/format/config/climate#weathersource)に指定すると、そのワールドの天候決定がアドオンに委ねられます。

```kotlin
import net.kyori.adventure.key.Key
import party.morino.moripafishing.api.core.world.weather.WeatherProvider
import party.morino.moripafishing.api.core.world.weather.WeatherSource

class MyWeatherSource : WeatherSource {
    override val key: Key = Key.key("myplugin", "realistic")

    // true の場合、コアが決定した天候をワールドに適用し DO_WEATHER_CYCLE を制御する
    override val managesWorldWeather: Boolean = true

    // true の場合、バニラの天候サイクル (DO_WEATHER_CYCLE) を有効に保つ
    override val usesVanillaWeatherCycle: Boolean = false

    override fun createProvider(worldId: FishingWorldId): WeatherProvider = MyWeatherProvider(worldId)
}

api.registerWeatherSource(MyWeatherSource())
```

`WeatherProvider`はpull型で、`getCurrentWeather`はメインスレッド・非同期スレッドの両方から呼ばれます。
Bukkit APIを直接読む実装は避け、イベント駆動でキャッシュしたスナップショットを返してください。

### dispose ライフサイクル

`WeatherProvider.dispose()`は、`weatherSource`の切替・設定リロード・ワールド削除・プラグイン無効化などで
プロバイダーが破棄されるときにコアが必ず呼び出します。BukkitのListenerやスケジューラタスクを
登録している実装は、ここで解除してください（デフォルトはno-op）。

### レジストリの挙動

- 同じキーで再登録すると上書きされます。組み込みキー（`moripafishing:internal` / `moripafishing:vanilla`）の上書きも可能ですが、警告ログが出力されます
- 登録元プラグインが無効化されると、そのプラグインが登録したソースは自動的に解除されます
- 解除・上書きされたソースを使用中のワールドは即座に再解決され、未登録キーは`moripafishing:internal`にフォールバックします
- `getWeatherSource(key)`で登録済みソースの取得、`getWeatherSourceKeys()`でキー一覧のスナップショットを取得できます

## イベント

MoripaFishingは、釣果・ワールド・天候・設定に関するBukkitイベントを発火します。
いずれも`moripafishing-api`に含まれ、通常のBukkitリスナーで購読できます。

| イベント | パッケージ | Cancellable | 発火タイミング |
|---|---|---|---|
| `AnglerFishCaughtEvent` | `event.fishing` | ✔ | 釣り上げの確定直前。キャンセルするとバニラの釣り上げごと取り消される。`setCaughtFish`で釣果を差し替え可能 |
| `AnglerFishCaughtResultEvent` | `event.fishing` | ✘ | 釣り上げ処理の完了後。通知・記録などの副作用はこちらを購読する |
| `AnglerJoinTeleportEvent` | `event.angler` | ✔ | 参加時テレポートの実行直前（WorldLifecycle Integration が発火）。`setDestination`で移動先を変更可能 |
| `FishingWorldCreateEvent` | `event.world` | ✘ | 釣りワールドの作成完了後 |
| `FishingWorldDeleteEvent` | `event.world` | ✔ | 釣りワールドの削除直前。キャンセルで削除中止 |
| `FishingWorldDeletedEvent` | `event.world` | ✘ | 釣りワールドの削除完了後 |
| `FishingWorldLoadEvent` | `event.world` | ✘ | 釣りワールドのロード完了後（起動時の読み込みと新規作成の両方） |
| `FishingWorldUnloadEvent` | `event.world` | ✘ | 釣りワールドのアンロード後（削除時・プラグイン無効化時） |
| `FishingWorldWeatherChangeEvent` | `event.world` | ✘ | 釣りワールドの実効天候の変化後。非同期スレッドから発火する場合がある（`isAsynchronous`で判別） |
| `ConfigReloadedEvent` | `event.config` | ✘ | `/mf reload`などによる設定再読み込みの完了後。設定由来の値をキャッシュしている場合はここで再取得する |

パッケージはいずれも`party.morino.moripafishing.`配下です（例: `party.morino.moripafishing.event.fishing.AnglerFishCaughtEvent`）。

### AnglerFishCaughtEvent

つりびとが魚を釣ったときに発生するイベントです。キャンセルや釣果の差し替えができます。
差し替えた釣果はアイテム化と後続の`AnglerFishCaughtResultEvent`に反映されます。

```kotlin
import party.morino.moripafishing.event.fishing.AnglerFishCaughtEvent

class MyListener : Listener {
    @EventHandler(ignoreCancelled = true)
    fun onFishCaught(event: AnglerFishCaughtEvent) {
        val angler = event.getAngler()
        val caughtFish = event.getCaughtFish()

        // 釣れた魚の情報を取得
        val fishId = caughtFish.fish
        val size = caughtFish.size
        val rarity = caughtFish.rarity
        val worth = caughtFish.worth

        // つりびとの情報
        val playerName = angler.getName()

        // イベントのキャンセル（釣り上げ自体を取り消す）
        // event.isCancelled = true

        // 釣果の差し替え
        // event.setCaughtFish(caughtFish.copy(worth = worth * 2))
    }
}
```

### AnglerFishCaughtResultEvent

釣り上げ処理が完了した後に発生するモニター用途のイベントです。
`AnglerFishCaughtEvent`がキャンセルされず、釣果のアイテム化まで完了した場合にのみ発火します。
アナウンスやログ記録などの副作用はこちらを購読してください。

```kotlin
import party.morino.moripafishing.event.fishing.AnglerFishCaughtResultEvent

class MyListener : Listener {
    @EventHandler
    fun onFishCaughtResult(event: AnglerFishCaughtResultEvent) {
        val caughtFish = event.getCaughtFish()
        // 確定した釣果に対する通知・記録など
    }
}
```

#### CaughtFish のフィールド

| フィールド | 型 | 説明 |
|---|---|---|
| `uniqueId` | `UUID` | 釣果の一意ID |
| `fish` | `FishId` | 魚のID |
| `size` | `Double` | 魚のサイズ |
| `rarity` | `RarityId` | レアリティID |
| `worth` | `Double` | 魚の価値 |
| `cfd` | `Double` | 累積分布関数（0〜1） |
| `angler` | `AnglerId` | つりびとID |
| `world` | `FishingWorldId` | 釣り場ワールドID |
| `timestamp` | `ZonedDateTime` | 釣った日時 |
