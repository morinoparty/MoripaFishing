釣り上げた魚の情報を表すデータクラスです（`party.morino.moripafishing.api.model.fish.CaughtFish`）。実装依存を持たない純粋なデータ構造で、現在は`AnglerFishCaughtEvent`と`AnglerFishCaughtResultEvent`でのデータ受け渡しと、釣果アイテムの生成に使用されます。

`@Serializable`が付与されているため、kotlinx.serializationでそのままJSON化できます。釣果履歴やランキングのように、Addon側で永続化やAPI返却を行いたい場合はこのクラスをそのまま利用できますが、Core自体は釣果をデータベースへ保存しません。

## フィールド

| フィールド | 型 | 説明 |
|---|---|---|
| `uniqueId` | `UUID` | 釣果の一意ID。UUIDv7（時系列順）で採番される |
| `fish` | `FishId` | 魚種ID |
| `size` | `Double` | 魚のサイズ。表示時は小数2桁に整形される |
| `rarity` | `RarityId` | レアリティID |
| `worth` | `Double` | 魚の価値 |
| `cfd` | `Double` | 累積分布関数値（0〜1） |
| `angler` | `AnglerId` | つりびとID |
| `world` | `FishingWorldId` | 釣り場ワールドID |
| `timestamp` | `ZonedDateTime` | 釣った日時。ISO8601形式の文字列としてシリアライズされる |

`uniqueId`は`UUIDSerializer`、`timestamp`は`ZonedDateTimeSerializer`によってシリアライズされます。`FishId`・`RarityId`・`FishingWorldId`はいずれも`value: String`を持つ値クラスで、専用のシリアライザによって文字列として直接シリアライズされます。生の識別子を取り出したい場合は`caughtFish.fish.value`のように`value`を参照してください。

`fish`・`rarity`・`world`の各IDは、それぞれ`moripa_fishing.fish.<id>.name`、`moripa_fishing.rarity.<id>.name`、`moripa_fishing.world.<id>.name`という翻訳キーに対応しています。各IDの`toTranslateKey()`でこのキーを取得できるため、閲覧者のロケールに応じた表示名を解決したい場合は、その戻り値を`Component.translatable`に渡してください。

## 生成

`CaughtFish.fromFish(fish, angler, world)`で`Fish`・`Angler`・`FishingWorld`から生成できます。`uniqueId`は時系列順のUUIDv7、`timestamp`は生成時点の`ZonedDateTime.now()`が自動的に設定されます。

```kotlin
val caughtFish = CaughtFish.fromFish(fish, angler, fishingWorld)
```

Bukkit側では`BukkitFishItem.create(caughtFish, locale)`によって実際の釣果アイテムを生成します。生成された`ItemStack`のPersistentDataContainerには、`moripa_fishing.fish`というキーで`uniqueId`が文字列として保存されるため、アイテムから釣果を逆引きできます。

## 利用例

`AnglerFishCaughtEvent`（釣り上げ確定前・キャンセルや差し替えが可能）と、
`AnglerFishCaughtResultEvent`（釣り上げ完了後・読み取り専用）から釣果データを取得できます。
通知や記録などの副作用には`AnglerFishCaughtResultEvent`を使用してください。

```kotlin
@EventHandler
fun onFishCaught(event: AnglerFishCaughtResultEvent) {
    val angler = event.getAngler()
    val caught = event.getCaughtFish()
    logger.info("つりびと: ${angler.getName()}, 魚: ${caught.fish.value}, サイズ: ${caught.size}, 価値: ${caught.worth}")
}
```

一方、`AnglerFishCaughtEvent`は`Cancellable`を実装しています。`setCancelled(true)`でキャンセルすると、Core側のリスナーが元の`PlayerFishEvent`もキャンセルするため、釣果アイテムは生成されず釣り自体が成立しません。特定条件で釣りを禁止したい場合に利用できます。

```kotlin
@EventHandler
fun onFishCaught(event: AnglerFishCaughtEvent) {
    if (isFishingBanned(event.getAngler())) {
        event.isCancelled = true
    }
}
```

また`AnglerFishCaughtEvent`では`setCaughtFish(caughtFish)`によって釣果そのものを差し替えられます。差し替えた内容はアイテム化と後続の`AnglerFishCaughtResultEvent`の両方に反映されます。ただし未登録の`FishId`へ差し替えるとアイテムを生成できず、その場合はバニラのドロップが維持されます。
