## 魚の価値計算

釣り上げたさかなの価値は、サイズをそのまま金額に換算するのではなく、そのさかな種の中でどれくらい大きい個体だったかという相対的な位置から決まります。このページでは価値が求まるまでの流れを説明し、サイズ範囲と数式を変えたときの傾向をシミュレーターで確かめられるようにしています。

数式そのものの書き方は [レアリティ](/docs/format/data/rarity) と [さかな](/docs/format/data/fish) の定義を参照してください。

## 価値が決まるまでの流れ

価値は次の 3 段階で決定されます。

1. さかな定義の `size` に書かれた最小値 `min` と最大値 `max` から、平均 `mid = (min + max) / 2`、標準偏差 `sd = (max - min) / 6` の正規分布を組み立てます。
2. その正規分布の累積分布関数に釣り上げたサイズを通し、0 から 1 の値を求めます。これが `<length_rate>` に代入される値で、`getCFD()` の戻り値そのものです。最小サイズ付近なら 0 に近く、最大サイズ付近なら 1 に近づきます。
3. 適用する数式の中の `<length_rate>` をその数値へ文字列置換し、数式として評価した結果を価値とします。

累積分布関数は Abramowitz-Stegun の近似式による誤差関数を使って計算しているため、厳密な正規分布の値とはわずかにずれます。表示される値は近似値だと考えてください。

## 適用される数式

数式は次の順で探索されます。

- さかな定義の `worthExpression` が `null` でなければ、それが最優先で使われます。
- さかな側に指定が無ければ、そのさかなのレアリティの `worthExpression` が使われます。レアリティ側の既定値は `"1"` なので、何も書かなければ価値は常に 1 になります。
- どちらにも数式が無い場合は `Worth expression not found for fish: <id>` という例外が発生します。

数式の評価には exp4j を使っています。四則演算のほか、`^` による累乗や `sin` / `cos` / `log` / `sqrt` などの関数が利用できます。

:::warning[注意]
累乗は `^` で書きます。`**` は exp4j の文法では解釈されず、設定として読み込むと評価に失敗します。同梱のレアリティ定義も `"(<length_rate> ^ 2) * 5 + 5"` のように `^` を使っています。
:::

## サイズの抽選と価値の分布

釣り上げるサイズ自体も、価値の計算と同じ `mid` と `sd` を持つ正規分布から抽選され、その結果を `min` から `max` の範囲へ丸めて決まります。範囲外に出た値は切り捨てられずに端の値へ寄せられるため、実測では最小値と最大値ちょうどの個体が理論上の正規分布より多く出ます。

算出された価値と累積分布関数の値は、釣果情報の `worth` と `cfd` として保存され、履歴やランキングの集計に使われます。

## シミュレーター

`サイズ` には、さかな定義の `size` に相当する最小値と最大値を入力します。`レアリティ` を切り替えると、そのレアリティに対応する数式で価値が再計算されます。表の `CDF 累積分布関数` の列が、そのサイズで `<length_rate>` に代入される値です。

:::info
シミュレーターはあくまで傾向をつかむためのものです。レアリティと数式はコンポーネントに直接書かれた初期状態のものなので、`rarity/*.json` を編集していると実際の計算と一致しません。また表示される数式はブラウザ上で評価する都合から累乗に `**` を使っていますが、設定ファイルに書くときは `^` に読み替えてください。さかな定義側の既定のサイズは 20.0 から 40.0 で、シミュレーターの初期値とは異なります。
:::
