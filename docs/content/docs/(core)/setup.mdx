サーバー管理者向けの MoripaFishing 導入・設定ガイドです。

## 動作要件

| 項目 | 要件 |
|------|------|
| Minecraft | Paper API 1.20 以降（開発とテストは Paper 26.1.2 で行っています） |
| サーバー | Paper / Purpur |
| Java | 21 以降 |
| 任意プラグイン | [WorldLifecycle 統合](/docs/integration/world-lifecycle)（釣りワールドの作成に必須）、[Weather 統合](/docs/integration/weather)、[Terra](https://modrinth.com/plugin/terra)（`terra` ジェネレーターを使う場合） |

:::warning[注意]
Spigot サーバーには対応していません。必ず Paper または Purpur を使用してください。
:::

## インストール

### 1. JAR ファイルの取得

以下のいずれかから最新の JAR ファイルをダウンロードしてください。

- [Modrinth](https://modrinth.com/plugin/moripafishing)
- [GitHub Releases](https://github.com/morinoparty/MoripaFishing/releases)

ダウンロードするファイルは `MoripaFishing_<バージョン>.jar` です。

### 2. プラグインの配置

ダウンロードした JAR ファイルをサーバーの `plugins/` ディレクトリに配置します。

```bash
plugins/
├── MoripaFishing_<バージョン>.jar
└── MoripaFishing-Integration-WorldLifecycle_<バージョン>.jar
```

:::note
釣りワールドを作成するには WorldLifecycle 統合プラグインが必要です。詳しくは [WorldLifecycle](/docs/integration/world-lifecycle) を参照してください。
:::

### 3. サーバーの起動

サーバーを起動すると、`plugins/MoripaFishing/` ディレクトリに設定ファイルが自動生成されます。

```bash
plugins/MoripaFishing/
├── config.json          # メイン設定
├── translation/         # 翻訳ファイル
│   ├── en_US.properties
│   └── ja_JP.properties
├── fish/                # 魚データ
├── world/               # ワールドデータ
└── rarity/              # レアリティデータ
```

なお、ジェネレーターの定義ファイルは Core ではなく WorldLifecycle 統合プラグイン側が管理しており、`plugins/MoripaFishing-Integration-WorldLifecycle/generator/` に配置されます。初回起動時に `terra` / `void` / `normal` の 3 つが既定として書き出されます。

起動時には `world/` 配下の JSON が読み込まれ、対応する Bukkit ワールドがまだロードされていない場合はその場で生成が試みられます。WorldLifecycle 統合が未導入だったり、指定されたジェネレーターが見つからなかったりした場合は、警告ログを出してそのワールドをスキップします。

## 初期設定

### config.json の基本設定

`plugins/MoripaFishing/config.json` を編集して基本設定を行います。
詳細は[コンフィグ](/docs/format/config)を参照してください。

主な設定項目:

- **デフォルトワールド**: `world.defaultId` でデフォルトの釣り場IDを指定します
- **タイムゾーン**: `world.defaultTimeZone` でタイムゾーンを設定します（デフォルト: `Asia/Tokyo`）
- **言語**: `defaultLocale` でプラグインの表示言語を設定します（`ja_JP` / `en_US`）

### 釣りワールドの作成

プラグイン導入後、釣り用のワールドを作成します。

:::warning[注意]
ワールドの作成には [WorldLifecycle 統合プラグイン](/docs/integration/world-lifecycle) が必要です。未導入の場合、ジェネレーター引数の解決に失敗してコマンドが実行できません。
:::

```
/mf world create <ワールドID> <generator>
```

`<generator>` には WorldLifecycle 統合が読み込んでいるジェネレーターの ID を指定します。既定では `terra` / `void` / `normal` の 3 つが利用できます（`terra` を使う場合は別途 Terra プラグインが必要です）。

例:
```
/mf world create fishing_world normal
```

なお `world` / `world_nether` / `world_the_end` はサーバー既定のワールド名として予約されており、釣りワールドの ID には使用できません。

ワールド作成後、以下のコマンドでスポーン地点やワールドサイズを設定できます。

| コマンド | 説明 |
|----------|------|
| `/mf world list` | 釣りワールドの一覧を表示 |
| `/mf world transfer <world> <player>` | プレイヤーを指定した釣りワールドへ移動 |
| `/mf world config set spawn <world>` | スポーン地点を現在位置に設定 |
| `/mf world config set center <world> [x] [z]` | ワールドの中心座標を設定（省略時は実行したプレイヤーの現在位置） |
| `/mf world config set size <world> <size>` | ワールドのサイズを設定 |
| `/mf world config view <world>` | ワールドの設定を確認 |
| `/mf world delete <world>` | 釣りワールドの登録を削除 |

:::note
`/mf world delete` はプラグインからの登録解除のみを行い、サーバー上のワールドフォルダは削除しません。不要になったデータは手動で削除してください。
:::

### 魚・レアリティの設定

`plugins/MoripaFishing/fish/` と `plugins/MoripaFishing/rarity/` に JSON ファイルを配置して、釣れる魚とレアリティを定義します。

- 魚データのフォーマット: [魚データ](/docs/format/data/fish)
- レアリティのフォーマット: [レアリティ](/docs/format/data/rarity)
- ジェネレーターのフォーマット: [ジェネレーター](/docs/format/data/generator)

## 権限設定

コマンドの権限は通常の Bukkit のパーミッションノードとして定義されているため、LuckPerms などの権限プラグインでそのまま設定できます。

| パーミッション | 説明 |
|----------------|------|
| `moripa_fishing.command.world` | `/mf` 以下のコマンド全体の基本権限 |
| `moripa_fishing.command.world.default` | ワールド一覧表示・プレイヤー移動・`/mf reload` |
| `moripa_fishing.command.world.create` | ワールドの作成 |
| `moripa_fishing.command.world.delete` | ワールドの削除 |
| `moripa_fishing.command.world.config` | ワールド設定の変更・表示 |

## 設定の反映

設定ファイルを変更した後は、以下のコマンドで再読み込みできます。

```
/mf reload
```

このコマンドはメイン設定、各釣りワールドの設定と状態、レアリティ、魚、翻訳ファイルの順に再読み込みを行い、進捗を実行者へ返します。

:::note
`/mf reload` の実行には `moripa_fishing.command.world.default` が必要です。reload 自体はワールド管理コマンドではありませんが、実装上このノードの配下に置かれています。
:::
