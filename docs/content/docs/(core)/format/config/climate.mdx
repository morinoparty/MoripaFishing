---
title: 天候
---

:::warning[注意]
この設定は、`config.json`及び、`world/<fishing_world_name>.json`に存在するデータであり、独立して存在するものではありません。
:::

## Format

```json
{
    "weatherSource": "moripafishing:internal",
    "applyWeatherEffects": true,
    "enableDayCycle": true,
    "constant" : {
        "weather": "SUNNY",
        "dayCycle": 1
    },
    "dayCycle" : {
        "offset": 0
    },
    "weather": {
       "weight": {
        "SUNNY": 4,
        "CLOUDY": 2,
        "RAINY": 2,
        "THUNDERSTORM": 1
       },
        "offset": 0,
        "interval": 8,
        "maxInclination": 30
    },
    "hashPepper": "pepper"
}
```

## 各設定の説明

### weatherSource

ワールドの天候を「誰が決定するか」を、名前空間キー (`namespace:value`) で指定します（デフォルト: `moripafishing:internal`）。指定したキーは `WeatherSourceRegistry` に登録された `WeatherSource` から解決されます。未登録のキーを指定した場合は `moripafishing:internal` にフォールバックします。

組み込みのソースは次の2つです。

- `moripafishing:internal`: プラグイン内蔵のランダマイザーが天候を決定し、Bukkit ワールドにも適用します。`DO_WEATHER_CYCLE` は `false` になります。
- `moripafishing:vanilla`: Bukkit のバニラ天候状態 (`hasStorm` / `isThundering`) を読み取り、魚の抽選条件にのみ反映します。`constant.weather` や `weather` の設定は無視され、`DO_WEATHER_CYCLE` は `true` のままになります。バニラ天候は `SUNNY` / `RAINY` / `THUNDERSTORM` の3値しか表現できないため、`CLOUDY` / `FOGGY` / `SNOWY` を条件に持つ魚はこのソースでは出現しません。

外部プラグインは `MoripaFishingAPI.registerWeatherSource` で独自キー（例: `myplugin:realistic`）の `WeatherSource` を登録できます。そのソースを使うには、`weatherSource` にそのキーを指定します。`WeatherSource.managesWorldWeather` が `false` のソースは MoripaFishing 側でワールドの天候を変更せず、魚の抽選条件にのみ反映されます。

### applyWeatherEffects

決定した天候を Bukkit ワールドへ実際に反映するかどうかを指定します（デフォルト: `true`）。

- `true`: `managesWorldWeather` が `true` のソース（`moripafishing:internal` など）が決定した天候をワールドに適用し、`DO_WEATHER_CYCLE` をソースに応じて制御します
- `false`: MoripaFishing はワールドの天候と `DO_WEATHER_CYCLE` を一切変更しません。天候は魚の抽選条件にのみ使用され、見た目の天候はバニラや他プラグインに委ねられます

天候カレンダー（`weather` の設定）は `false` でも動作し続けるため、「抽選条件は内蔵ランダマイザーで決めつつ、演出は別プラグインに任せる」構成をワールド単位で実現できます。

### ユースケース別の設定

`weatherSource` と `applyWeatherEffects` を組み合わせることで、天候の「決定」と「適用」を独立して制御できます。

| # | 天候の決定 | 見た目への適用 | 設定 |
|---|---|---|---|
| 1 | 内蔵ランダマイザー | する | `weatherSource: "moripafishing:internal"` + `applyWeatherEffects: true`（デフォルト） |
| 2 | 内蔵ランダマイザー | しない | `weatherSource: "moripafishing:internal"` + `applyWeatherEffects: false` |
| 3 | 外部プラグイン | する | `weatherSource: "<外部キー>"`（`managesWorldWeather = true` なソース）+ `applyWeatherEffects: true` |
| 4 | 外部プラグイン | しない | `weatherSource: "<外部キー>"` + `applyWeatherEffects: false`（または `managesWorldWeather = false` なソース） |
| 5 | バニラ天候の読み取り | しない | `weatherSource: "moripafishing:vanilla"`（`applyWeatherEffects` は影響しない） |

いずれの構成でも、魚の抽選条件への天候の反映（消費側）は変わりません。決定と適用だけが差し替わります。

:::info
実効天候が変化すると `FishingWorldWeatherChangeEvent` が発火します。イベントの一覧は [API利用ガイド](/docs/addons/api-guide#イベント) を参照してください。
:::

### enableDayCycle

プラグインによる時間同期を有効にするかどうかを指定します（デフォルト: `true`）。

- `true`: `DO_DAYLIGHT_CYCLE`を`false`に設定し、プラグインがリアルタイム同期または固定時間を管理します（従来の動作）
- `false`: `DO_DAYLIGHT_CYCLE`を`true`に設定し、バニラの昼夜サイクルに委ねます。`constant.dayCycle`や`dayCycle`の設定は無視されます

:::info
この設定はワールドの状態更新（デフォルト60秒間隔）ごとに再評価されるため、`/mf reload`後に自動で反映されます。
:::

### constant

天候を固定する場合の設定です。

- `constant.weather`: 固定する天候を指定します。nullの場合は天候が変化します（デフォルト: null）
- `constant.dayCycle`: 固定する時間を指定します。nullの場合は天候が変化します（デフォルト: null）

### weather

天候が変化する場合の設定です。

- `weather.weight`: 各天気の出現確率を指定します
- `weather.offset`: 天気の更新タイミングのオフセット（時間）を指定します
- `weather.interval`: 天気の更新間隔（時間）を指定します
- `weather.maxInclination`: 天気の変化の最大傾斜（度）を指定します

### hashPepper

天気予測のハッシュ生成に使用するペッパー文字列を指定します。

### dayCycle

日周期の設定です。設定されたタイムゾーンに従って日周期を制御します。

日時の更新は、`config.json`の`world.refreshInterval`で指定された間隔で定期的に実行され、現実世界の日時に同期して日周期を設定します。

- `dayCycle.offset`: 日周期のオフセット（時間）を指定します。`config.json`の`world.defaultTimeZone`に基づいて取得された値に対して、`dayCycle.offset`を加算した値が使用されます。
