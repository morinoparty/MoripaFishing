MoripaFishingは以下のディレクトリ配置をサポートしています。

## ディレクトリ

```bash
plugins/MoripaFishing/
├── config.json
├── translation/
│   ├── en_US.properties
│   └── ja_JP.properties
├── fish/
│    └── <rarity_id>/
│        └── <fish_id>.json
├── world/
│    └── <world_id>.json
└── rarity/
     └── <rarity_id>.json
```

各ディレクトリは`PluginDirectory`インターフェース経由で取得され、参照された時点で存在しなければ自動的に作成されます。そのため初回起動後は上記の構成が揃った状態になります。

`config.json`はファイルが存在しない場合、あるいは0バイトの場合に、`ConfigData`の既定値で自動生成されます。ルートディレクトリに置かれる設定ファイルはこの1つだけです。

## ファイル名とID

`world/`と`rarity/`のファイル名は、表示名ではなくそれぞれのID（`FishingWorldId`、`RarityId`）で解決されます。`fish/`のサブディレクトリ名もレアリティIDです。表示名を変更してもファイル名には影響しないので、IDと一致した名前を保ってください。

各ディレクトリで読み込まれるのは拡張子が`json`のファイルのみです。`fish/`はルート直下とその1階層下のサブディレクトリまでしか走査せず、それより深い階層には再帰しません。

## 魚ファイルの自動整理

`fish/`直下に置いた`<fish_id>.json`も読み込まれますが、読み込み時にファイル内の`rarity`の値と同じ名前のサブディレクトリへ自動的に移動されます。そのため直下に置いたファイルは起動後に`fish/<rarity_id>/`配下へ整理されます。手動で分類する必要はありませんが、配置したパスがそのまま維持されるわけではない点に注意してください。

## プラグイン側からの書き戻し

`world/<world_id>.json`は、スポーン地点やワールドボーダーの変更などに応じてプラグイン側から上書きされます。サーバー稼働中にファイルを直接編集すると、書き戻しによって変更が失われることがあります。手編集はサーバー停止中に行うか、編集後に`/mf reload`で反映させてください。

## ジェネレーターの定義

ワールドジェネレーターの定義はCore本体ではなく、`world-lifecycle`統合プラグイン側で管理されます。配置先はそのプラグインのデータフォルダ配下です。

```bash
plugins/MoripaFishing-Integration-WorldLifecycle/
└── generator/
     └── <generator_id>.json
```

初回起動時に`terra`、`void`、`normal`の3つの既定ジェネレーターが書き出されます。Core単体では`plugins/MoripaFishing/generator/`は使われません。
