## 補足

すべてのサブコマンドは、クラス全体に設定された `moripa_fishing.command.world` を前提としています。各サブコマンドの権限は、この基本権限に加えて必要になるものです。

### ワールドの作成について

`create` はジェネレーターの指定が必須で、省略できません。ジェネレーター名は WorldLifecycle 統合から解決されるため、統合プラグインが導入されていない場合は `WorldLifecycle integration is not installed.` として失敗します。また、`WorldManager` の実装が既定の `WorldManagerImpl` でない場合も作成を中断し、その旨をメッセージで通知します。

### ワールドの削除について

`delete` は釣りワールドの管理対象からの削除のみを行い、ディスク上のワールドフォルダーは残したままにします。実行後のメッセージにも、フォルダーが削除されていないことが明示されます。

### 設定コマンドについて

`config set spawn` は実行者の現在位置をスポーン座標として保存するため、プレイヤーからのみ実行できます。`config set center` は座標を指定して実行することもできますが、`x` と `z` を省略した場合は実行者の現在位置が使われるため、その形式ではプレイヤー専用となります。

`config view` はワールド設定を整形せず、`WorldDetailConfig` を JSON にエンコードした内容をそのまま出力します。
