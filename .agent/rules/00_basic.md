## 重要

- このリポジトリは `MoripaFishing` のモノレポです。
- 実装前に、必ず現在のコード・ディレクトリ構成・既存ルールを確認し、別プロジェクトの前提で判断しないでください。
- 変更は既存実装の方針に合わせ、小さく安全に進めてください。
- なお現状は開発版のため、今後に負債を残さないよう破壊的変更を行ってもかまいません。
- また、開発段階でのマイグレーション等の実装は夫妻となるのでしないでください。
- コンテキスト不足で誤実装の可能性が高い場合のみ、短く確認してください。

## 作業開始準備

- 作業開始時に `git status` でワークツリーを確認してください。
- 指示内容と無関係な変更が多い場合は、巻き込まずに進められるかを先に判断してください。
- 既存の変更が今回の修正と衝突する場合のみ、作業前にユーザーへ確認してください。

## 作業の進め方

- まず関連ファイルを読み、実際の構成に沿って変更してください。
- Kotlin 側の変更では `api` と `bukkit` の責務分離を崩さないでください。
- ドキュメント変更が必要な場合は `docs/` 配下の Fumadocs 構成に合わせて更新してください。
- 作業後は可能な範囲で `task test` または `task check`、必要なら `task build` を実行して確認してください。
